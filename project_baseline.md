# 项目基线规则

## 1. 项目名称
个人技术博客

## 2. 版本规则
语义化版本（v1.0.0为初始上线版本）

## 3. 唯一开发依据
最新版策划PRD文档 + 架构设计文档

## 4. 文档命名规范
所有文档必须带版本号，如 blog_prd_v1.0.0.md

## 5. 权责规则
- **planner**：仅负责需求与PRD
- **architect**：仅负责架构设计
- **coder**：仅负责代码实现

## 6. 禁止规则
- 无文档支撑的功能/修改一律禁止实现
- 禁止跨智能体越界修改上游输出内容

## 7. 需求基线锁定

**锁定文档**：`/docs/planner/blog_prd_v1.0.0.md`  
**锁定日期**：2026-02-15  
**锁定状态**：🔒 已锁定

### 基线规则
1. 本次开发的唯一需求依据为锁定的PRD文档，无正规变更流程，禁止任何修改
2. Architect与Coder智能体仅可读取该文档，不可修改
3. 所有架构设计、代码开发必须100%对齐该文档，无偏离
4. 任何需求变更必须走正规变更流程：提报 → 评审 → 更新PRD版本 → 重新锁定

3. 所有代码开发必须100%对齐该文档，无偏离、无越界
